# SocialHub 取扱説明書

Discord × X(旧Twitter)を融合したリアルタイムSNSアプリの利用ガイドです。
このファイルは配布ZIPのルート(`social-app/MANUAL.md`)に含まれています。

---

## 1. はじめに使う前に(重要)

- **アカウント登録・ログインが必須です。** サイトを初めて開いたとき、未ログインの状態でどのページを開いても、ログイン画面(`/login`)へ強制的に案内されます。トップページを直接叩いても、ログアウト直後でも同じです。これは `src/middleware.ts` が全ページ・全APIをガードしているためで、公開されているのは以下のみです。
  - `/login`(ログイン)
  - `/register`(新規登録)
  - `/support`(パスワード再発行申請フォーム)
  - `/api/auth/*`, `/api/register`, `/api/support/*`, `/api/health`(認証・登録・復旧・死活監視用API)
- メールアドレスは不要です。**ユーザー名(英数字と`_`のみ、3〜20文字)とパスワード(6文字以上)だけ**でアカウントを作成できます。
- パスワードを忘れた場合は `/support` から再発行を申請し、管理者(`admin`)が承認・一時パスワードを発行します。

### 初期管理者アカウント
| ユーザー名 | 初期パスワード |
|---|---|
| `admin` | `admin1234`(環境変数 `ADMIN_INITIAL_PASSWORD` で変更可) |

⚠️ 本番公開前に **必ず設定画面からパスワードを変更してください。**

## 2. 画面モードについて(X風 / Discord風 / ハイブリッド)

設定画面(`/settings`)から3つの表示モードを切り替えられます。

| モード | 内容 |
|---|---|
| `discord` | 左側にサーバーレール(参加サーバー一覧)を表示。Discordのような多人数チャット中心のUI |
| `x` | サーバーレールを隠し、タイムライン中心のX(Twitter)風UI |
| `hybrid`(既定) | 両方の要素を表示 |

### Xモードで見ている間にDiscordのメッセージが来たら?
Xモードにしていても、参加しているDiscordサーバーのチャンネルに新しいメッセージが届くと、
1. **ブラウザ通知**(OS標準の通知。初回にブラウザへの通知許可が必要です)にタイトルとして
   `Discord通知 — サーバー名 ・ #チャンネル名` が表示され、本文に「誰が」「何を」書いたかが出ます。
2. アプリ画面右上にも **一時的なトースト表示**(6秒間)が出て、「〇〇さんから新着メッセージ:本文抜粋」を確認できます。
3. 通知ベル(`/notifications`)にも記録が残り、後からクリックすると該当のサーバー・チャンネルへ直接ジャンプします。

これにより、Xモードで作業中でもDiscordチャンネルのメッセージを見落としません。

## 3. 主な機能

### 投稿(X風タイムライン)
- ホーム(`/home`)から最大500文字・画像4枚までの投稿が可能。
- `@ユーザー名` でメンションすると相手に通知が届きます。
- いいね・リアクション(絵文字)・リツイート・返信(スレッド)に対応。返信は投稿詳細ページ(`/post/[id]`)で確認できます。
- 投稿は **データベース(PostgreSQL)に保存**され、ページを開いている全員にリアルタイム反映されます(Socket.IOによる即時配信 + APIによる正規の永続化)。
- 「すべて」「フォロー中」タブでタイムラインを切り替え可能。
- `/explore` でユーザー検索・全体の投稿一覧を確認できます。

### Discordサーバー(チャンネルチャット・ボイス通話)
- サーバーレールからサーバーを新規作成、または `/servers/discover` で公開サーバーを探して参加。
- テキストチャンネル・ボイスチャンネルを追加できます(オーナーのみ)。
- ボイスチャンネルはWebRTCによる複数人同時通話に対応(映像ON/OFF、マイクミュート可)。
- サーバーメンバー以外はチャンネルの閲覧・投稿ができないようアクセス制御されています。
- オーナーはサーバー自体を削除できます(取り消し不可のため確認ダイアログあり)。

### ダイレクトメッセージ(DM)
- `/dm` から会話一覧、`/dm/[userId]` で1対1チャット。
- 画像添付、絵文字リアクション、タイピング中表示に対応。
- 1対1の音声・ビデオ通話をDM画面から開始できます。

### 通知
- いいね・リアクション・リツイート・フォロー・返信・メンション・DM・**Discordチャンネルの新着メッセージ**まで、すべて `/notifications` に一覧表示されます。
- 各通知をクリックすると、発生源(投稿・チャンネル・DM・プロフィール)へ直接移動します。
- 未読件数はサイドバーのベルアイコンにバッジ表示されます。

### プロフィール・関係管理
- プロフィール編集(表示名・自己紹介・アバター・バナー)。
- フォロー / フォロー解除、ミュート、ブロック(ブロックするとDM・フォロー関係も解除されます)。
- ブロック・ミュート中のユーザー一覧は `/settings/blocked` から確認・解除できます。

### テーマ
- ダーク・ライト・ミッドナイト・オーシャン・フォレスト・ローズの6テーマ + アクセントカラーを設定画面から選択可能。

## 4. データの永続性について(アップデートしても投稿は消えません)

- すべての投稿・メッセージ・通知・ユーザー情報はPostgreSQLデータベースに保存されます。アプリのコードを更新して再デプロイしても、**データベース自体は別サービス(Render PostgreSQL)として存続するため投稿は消えません。**
- デプロイ時のビルドコマンドは `npx prisma migrate deploy` を使用しています。これは「テーブル構造の差分だけを安全に反映する」コマンドで、既存データを削除する `prisma db push --accept-data-loss` とは異なり、**既存の投稿・ユーザーデータを保持したままスキーマ更新ができます。**
- 今後スキーマ(`prisma/schema.prisma`)を変更する場合は、ローカルで `npx prisma migrate dev --name 変更内容` を実行し、生成された `prisma/migrations/` フォルダをコードと一緒にコミット・デプロイしてください。これにより本番DBにも同じ変更が安全に反映されます。
- 初期化用の `prisma/seed.js` は、`admin` ユーザーが既に存在する場合は何もしないので、再デプロイのたびに実行されても問題ありません。

## 5. デプロイ方法(Render)

`DEPLOY_GUIDE.md` に詳細手順がありますが、要点は以下です。

1. GitHubにこのプロジェクトをpushする。
2. [Render](https://dashboard.render.com/) でBlueprint(`render.yaml`)から一括作成、またはWeb Service + PostgreSQLを個別作成。
3. 環境変数を設定(`DATABASE_URL` はDB作成時に自動連携、`NEXTAUTH_SECRET` は自動生成、`NEXTAUTH_URL` と `NEXT_PUBLIC_SOCKET_URL` は発行されたURLに合わせて設定)。
4. 初回デプロイ完了後、`admin` / 初期パスワードでログインし、必ずパスワードを変更。
5. 以後のコード更新は、GitHubにpush → Renderが自動で再ビルド・再デプロイ(データは保持されます)。

## 6. ローカルでの動かし方

```bash
npm install
cp .env.example .env   # DATABASE_URL などを編集
npx prisma migrate deploy   # または初回のみ migrate dev
npm run db:seed
npm run dev
```

`npm run dev` は Next.js と Socket.IOサーバーを同時起動します(`concurrently`)。

## 7. よくある質問・使い方のコツ

- **投稿・返信の送信**: `Ctrl + Enter` でも送信できます。
- **チャット送信**: `Enter` で送信、`Shift + Enter` で改行。
- **画像**: 投稿・チャンネル・DMいずれも最大4枚まで添付可(自動圧縮されます)。
- **通知が来ない**: ブラウザの通知許可がブロックされていないか確認してください(初回アクセス時にダイアログが出ます)。
- **パスワードを忘れた**: `/support` から再発行申請 → 管理者が対応。
- **他人をブロックしたい**: プロフィール画面のメニューから「ブロック」。DM・フォロー関係も自動解除されます。
